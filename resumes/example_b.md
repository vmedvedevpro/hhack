# Resume B (example)

Second placeholder resume. Use this slot if you have a second resume
targeting a different role track (e.g. one for backend roles and one
for AI/ML roles). Point `RESUME_B_PATH` at it.

If you only have one resume, set `RESUME_B_PATH` to the same path as
`RESUME_A_PATH` — the matcher will still work but every job will be
evaluated against the same content twice.

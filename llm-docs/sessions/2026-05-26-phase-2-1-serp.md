# Phase 2.1 — paginated SERP crawl

## Что обсудили

В первом живом прогоне Phase 2 (см. `2026-05-26-phase-2-feed.md`)
выяснилось, что главная hh.ru — тизер на ~5 карточек, реальный feed
живёт по ссылке «Посмотреть N вакансий» с `resume=<id>` в query.
Открытые вопросы перед началом сессии:

- **Тизер vs SERP.** Парсим только SERP — `&page=0` уже содержит
  карточки тизера, два парсинга на одной странице только наплодят
  дублей и путаницу в дампах.
- **Навигация.** Через `page.goto(url + "&page=N")`. Детерминированно,
  не зависит от вёрстки пагинатора и сохраняет hh-овые query-параметры
  (`resume`, `hhtmFromLabel`, `hhtmFrom`) байт-в-байт.
- **CLI флаг.** Roadmap прямо просил `--max-scrolls` → `--max-pages`,
  сделали.

Решения зафиксированы в [D-021](decisions.md#d-021--2026-05-26--phase-21-drops-the-teaser-serp-is-the-only-source).

## Что сделали по коду

- `src/hhack/integrations/hh/urls.py` — добавлены `extract_resume_id`
  и `search_url_with_page`. Обе через `urllib.parse`, не регексом:
  URL HH тащит десяток параметров, и важно сохранить их при подмене
  `page=N`.
- `src/hhack/integrations/hh/feed.py` — переписан:
  - `collect_search_buttons(page)` — JS-эвалюатор, собирает
    `a[data-qa="applicant-index-search-all-results-button"]` с
    дедупом по `href`.
  - `harvest_cards(page, *, resume_id=None)` — тот же JS, но теперь
    проставляет `feed_resume_hint=resume_id` через Python, а не через
    DOM. JS-харвестер избавлен от попыток выудить hint из карточки
    (D-021 / D-019 — hint живёт в URL источника).
  - `discover_new_cards(page, repo, *, max_pages, dump_dir)` — внешний
    цикл по кнопкам с главной, внутренний цикл по `page=0..N` до
    первого known `hh_id` или потолка `max_pages`. Сквозной
    `feed_position` 1..M по всему прогону.
  - Diagnostics: на каждый прогон пишутся HTML главной, HTML page=0
    каждого SERP, и один общий JSON по всем собранным карточкам.
    Имена файлов получили суффикс-метку
    (`feed-<ts>-main.html`, `feed-<ts>-serp-resume-<id>.html`).
- `src/hhack/workers/feed.py` — CLI флаг `--max-scrolls` →
  `--max-pages`, передача параметра в `discover_new_cards`,
  обновлённый docstring модуля.
- `tests/integrations/hh/test_urls.py` — четыре новых теста на
  `extract_resume_id` и `search_url_with_page` (включая replace
  существующего `page=` параметра).
